## DNS Crisis: www.bithues.com

**Problem:** www.bithues.com returning HTTP 405 from GitHub. Spent 45+ minutes misdiagnosing.

**Root cause (actual):** GitHub Pages settings didn't have `www.bithues.com` added as a custom domain — apex domain was configured but not www. DNS was actually correct (verified from 1.1.1.1 and 8.8.8.8).

**What I did wrong:** Confused zone-level DNS records with registry-level NS delegation. Told Mike to delete A records that didn't exist. Didn't check GitHub Pages settings first.

**DNS is correct — just GitHub Pages config needed:**
- `dig ns www.bithues.com` → returns Cloudflare (correct, not Afternic)
- `dig www.bithues.com +short` → `michaelbacotti.github.io` + GitHub IPs ✅
- Public resolvers (1.1.1.1, 8.8.8.8) → CNAME chain working ✅
- GitHub Pages still showing warning → stale cache on GitHub's side

**Lesson:** When GitHub says "DNS check unsuccessful" → FIRST check if www is added in GitHub Settings → Pages → Custom domains. DNS being correct at 1.1.1.1 means the problem is GitHub's config, not DNS.

**Created:** `skills/dns-diagnostic/SKILL.md`
